查看域名的NS

$ dig ns xxx.com

; <<>> DiG 9.18.39-0ubuntu0.22.04.2-Ubuntu <<>> ns xxx.com
;; global options: +cmd
;; Got answer:
;; ->>HEADER<<- opcode: QUERY, status: NOERROR, id: 35928
;; flags: qr rd ra; QUERY: 1, ANSWER: 2, AUTHORITY: 0, ADDITIONAL: 1

;; OPT PSEUDOSECTION:
; EDNS: version: 0, flags:; udp: 65494
;; QUESTION SECTION:
;xxx.com.      IN  NS

;; ANSWER SECTION:
xxx.com.   86400 IN  NS  duke.ns.cloudflare.com.
xxx.com.   86400 IN  NS  tia.ns.cloudflare.com.

;; Query time: 98 msec
;; SERVER: 127.0.0.53#53(127.0.0.53) (UDP)
;; WHEN: Fri Feb 27 09:23:24 CST 2026
;; MSG SIZE  rcvd: 88

查看域名的A记录

$ dig A xxx.com

; <<>> DiG 9.18.39-0ubuntu0.22.04.2-Ubuntu <<>> A xxx.com
;; global options: +cmd
;; Got answer:
;; ->>HEADER<<- opcode: QUERY, status: NOERROR, id: 19752
;; flags: qr rd ra; QUERY: 1, ANSWER: 2, AUTHORITY: 0, ADDITIONAL: 1

;; OPT PSEUDOSECTION:
; EDNS: version: 0, flags:; udp: 65494
;; QUESTION SECTION:
;xxx.com.      IN  A

;; ANSWER SECTION:
xxx.com.   14  IN  A 23.54.118.38
xxx.com.   14  IN  A 23.54.118.49

;; Query time: 280 msec
;; SERVER: 127.0.0.53#53(127.0.0.53) (UDP)
;; WHEN: Fri Feb 27 10:20:52 CST 2026
;; MSG SIZE  rcvd: 69

DNS 查询工具，可以直接查询指定域名的A记录

$ dig @duke.ns.cloudflare.com xxx.com

; <<>> DiG 9.18.39-0ubuntu0.22.04.2-Ubuntu <<>> @duke.ns.cloudflare.com xxx.com
; (6 servers found)
;; global options: +cmd
;; Got answer:
;; ->>HEADER<<- opcode: QUERY, status: NOERROR, id: 17763
;; flags: qr aa rd; QUERY: 1, ANSWER: 2, AUTHORITY: 0, ADDITIONAL: 1
;; WARNING: recursion requested but not available

;; OPT PSEUDOSECTION:
; EDNS: version: 0, flags:; udp: 1232
;; QUESTION SECTION:
;xxx.com.      IN  A

;; ANSWER SECTION:
xxx.com.   60  IN  A 23.54.118.38
xxx.com.   60  IN  A 23.54.118.49

;; Query time: 77 msec
;; SERVER: 172.64.33.110#53(duke.ns.cloudflare.com) (UDP)
;; WHEN: Fri Feb 27 10:25:14 CST 2026
;; MSG SIZE  rcvd: 69

可以理解为：“我直接问 Cloudflare，xxx.com 的源站现在到底是哪几台服务器”， 如果上面的 dig 返回的结构和下面这个不一致，说明本地的DNS有问题(DNS劫持)
