Memcached 使用手册

简介

Memcached 是一个高性能的分布式内存对象缓存系统,用于加速动态 Web 应用程序,减轻数据库负载。它通过在内存中缓存数据和对象来减少数据库访问次数。

核心特性

  • 基于内存的键值存储系统
  • 简单的协议设计(文本协议和二进制协议)
  • 分布式架构支持
  • LRU(最近最少使用)淘汰策略
  • 支持多线程
  • 高性能,低延迟

安装与配置

Linux 安装

Ubuntu/Debian

sudo apt-get update
sudo apt-get install memcached libmemcached-tools

CentOS/RHEL

sudo yum install memcached libmemcached

从源码编译

wget http://memcached.org/latest
tar -zxvf memcached-1.x.x.tar.gz
cd memcached-1.x.x
./configure && make && sudo make install

启动 Memcached

基本启动

memcached -d -m 64 -p 11211 -u memcache -l 127.0.0.1

参数说明

  • -d: 以守护进程方式运行
  • -m: 分配的内存大小(MB),默认 64MB
  • -p: 监听端口,默认 11211
  • -u: 运行用户
  • -l: 监听地址
  • -c: 最大并发连接数,默认 1024
  • -t: 线程数,默认 4
  • -P: 保存 PID 的文件路径

配置文件方式(systemd)

编辑 /etc/memcached.conf:

# 内存限制
-m 512

# 监听端口
-p 11211

# 监听地址
-l 127.0.0.1

# 最大连接数
-c 2048

# 线程数
-t 8

# 用户
-u memcache

启动服务:

sudo systemctl start memcached
sudo systemctl enable memcached

基本操作

使用 telnet 连接测试

telnet localhost 11211

存储命令

set - 设置键值

set key flags exptime bytes
value

# 示例
set mykey 0 3600 5
hello
STORED

参数说明:

  • key: 键名
  • flags: 标志位(客户端用于标识数据类型)
  • exptime: 过期时间(秒),0 表示永不过期
  • bytes: 数据字节数
  • value: 实际数据

add - 仅当键不存在时添加

add newkey 0 3600 5
world
STORED

replace - 仅当键存在时替换

replace mykey 0 3600 7
goodbye
STORED

append - 追加数据到现有值

append mykey 0 3600 6
 world
STORED

prepend - 前置数据到现有值

prepend mykey 0 3600 6
hello 
STORED

读取命令

get - 获取单个或多个键

get mykey
VALUE mykey 0 5
hello
END

get key1 key2 key3

gets - 获取带 CAS 标识的值

gets mykey
VALUE mykey 0 5 123
hello
END

删除命令

delete - 删除键

delete mykey
DELETED

计数器操作

incr - 增加数值

set counter 0 0 1
5
STORED

incr counter 3
8

decr - 减少数值

decr counter 2
6

CAS(Compare And Swap)操作

# 获取 CAS 值
gets mykey
VALUE mykey 0 5 123
hello
END

# 使用 CAS 更新
cas mykey 0 3600 5 123
world
STORED

统计命令

stats - 服务器统计信息

stats
STAT pid 1234
STAT uptime 3600
STAT time 1234567890
STAT version 1.6.21
...
END

stats items - 项目统计

stats items

stats slabs - 内存分配统计

stats slabs

stats sizes - 键大小分布

stats sizes

其他命令

flush_all - 清空所有数据

flush_all
OK

# 延迟清空(秒)
flush_all 60
OK

version - 查看版本

version
VERSION 1.6.21

quit - 断开连接

quit

高级特性

二进制协议

Memcached 支持更高效的二进制协议,相比文本协议具有以下优势:

  • 更少的解析开销
  • 更紧凑的数据格式
  • 支持更多数据类型
  • 内置 CAS 支持

大多数客户端库默认使用二进制协议。

SASL 认证

启用 SASL 认证增强安全性:

memcached -S -vv

配置用户名和密码:

echo "user:password" > /etc/sasldb2
saslpasswd2 -a memcached -c username

TLS/SSL 支持

Memcached 1.5.13+ 支持 TLS 加密:

memcached -Z -o ssl_chain_cert=/path/to/cert.pem \
          -o ssl_key=/path/to/key.pem

扩展存储引擎

Memcached 支持外部存储引擎,可以将不常用数据存储到磁盘:

memcached -o ext_path=/path/to/storage:1G

编程语言客户端

Python

import memcache

# 连接
mc = memcache.Client(['127.0.0.1:11211'])

# 设置
mc.set('key', 'value', time=3600)

# 获取
value = mc.get('key')

# 删除
mc.delete('key')

# 批量操作
mc.set_multi({'key1': 'value1', 'key2': 'value2'})
values = mc.get_multi(['key1', 'key2'])

PHP

<?php
$memcache = new Memcached();
$memcache->addServer('127.0.0.1', 11211);

// 设置
$memcache->set('key', 'value', 3600);

// 获取
$value = $memcache->get('key');

// 删除
$memcache->delete('key');

// CAS 操作
$cas = null;
$memcache->get('key', null, $cas);
$memcache->cas($cas, 'key', 'new_value');
?>

Java

// 关闭

Node.js

Go

性能优化

内存分配优化

调整 chunk 大小

memcached -f 1.25  # 增长因子,默认 1.25

预分配内存

memcached -L  # 使用大内存页

连接池配置

客户端应使用连接池以减少连接开销:

# Python 示例
mc = memcache.Client(
    ['127.0.0.1:11211'],
    debug=0,
    socket_timeout=3,
    server_max_value_length=1024*1024
)

键设计最佳实践

  1. 使用简短但有意义的键名
  2. 使用命名空间前缀(如 user:1234)
  3. 避免使用过长的键(最大 250 字节)
  4. 键名使用 ASCII 字符,避免特殊字符

数据压缩

对于大数据,客户端可以自动压缩:

mc.set('large_data', large_object, compress_threshold=1024)

分布式策略

一致性哈希

使用一致性哈希算法分布数据到多个节点:

from memcache import Client

servers = [
    '192.168.1.1:11211',
    '192.168.1.2:11211',
    '192.168.1.3:11211'
]

mc = Client(servers)

监控与维护

监控指标

关键指标

  • cmd_get: GET 命令总数
  • cmd_set: SET 命令总数
  • get_hits: 命中次数
  • get_misses: 未命中次数
  • evictions: 驱逐数量
  • bytes: 当前存储字节数
  • curr_connections: 当前连接数
  • total_connections: 总连接数

计算缓存命中率

命中率 = get_hits / (get_hits + get_misses) * 100%

监控工具

memcached-tool

memcached-tool localhost:11211 stats
memcached-tool localhost:11211 display

使用脚本监控

#!/bin/bash
echo "stats" | nc localhost 11211 | grep -E "cmd_get|get_hits|get_misses"

日志分析

启用详细日志:

memcached -vv

常见问题排查

内存不足

  • 检查 evictions 指标
  • 增加内存分配:-m 参数
  • 优化数据过期策略

连接数过多

  • 检查 curr_connections
  • 增加最大连接数:-c 参数
  • 检查客户端连接池配置

性能下降

  • 检查 CPU 使用率
  • 增加线程数:-t 参数
  • 优化网络配置

最佳实践

缓存策略

Cache-Aside(旁路缓存)

def get_user(user_id):
    # 先查缓存
    user = mc.get(f'user:{user_id}')
    if user is None:
        # 缓存未命中,查数据库
        user = db.query(f'SELECT * FROM users WHERE id={user_id}')
        # 写入缓存
        mc.set(f'user:{user_id}', user, 3600)
    return user

Write-Through(写穿)

def update_user(user_id, data):
    # 先更新数据库
    db.update(f'UPDATE users SET ... WHERE id={user_id}')
    # 同步更新缓存
    mc.set(f'user:{user_id}', data, 3600)

Write-Behind(写回)

def update_user(user_id, data):
    # 先更新缓存
    mc.set(f'user:{user_id}', data, 3600)
    # 异步更新数据库
    queue.add_task('update_db', user_id, data)

过期时间设置

# 短期数据(会话)
mc.set('session:123', data, 1800)  # 30分钟

# 中期数据(用户信息)
mc.set('user:456', data, 86400)  # 24小时

# 长期数据(配置)
mc.set('config:app', data, 604800)  # 7天

缓存预热

def warmup_cache():
    # 预加载热点数据
    hot_users = db.query('SELECT * FROM users WHERE is_hot=1')
    for user in hot_users:
        mc.set(f'user:{user.id}', user, 3600)

防止缓存雪崩

设置随机过期时间

import random

expire_time = 3600 + random.randint(0, 300)
mc.set('key', value, expire_time)

使用互斥锁

def get_data_with_lock(key):
    data = mc.get(key)
    if data is None:
        lock_key = f'lock:{key}'
        if mc.add(lock_key, 1, 10):  # 获取锁
            try:
                data = db.query(...)
                mc.set(key, data, 3600)
            finally:
                mc.delete(lock_key)  # 释放锁
        else:
            time.sleep(0.1)
            return get_data_with_lock(key)
    return data

防止缓存穿透

缓存空值

user = db.query(f'SELECT * FROM users WHERE id={user_id}')
if user is None:
    mc.set(f'user:{user_id}', 'NULL', 300)  # 缓存5分钟
else:
    mc.set(f'user:{user_id}', user, 3600)

布隆过滤器

from pybloom import BloomFilter

bf = BloomFilter(capacity=1000000, error_rate=0.001)

# 初始化
for user_id in all_user_ids:
    bf.add(user_id)

# 查询
def get_user(user_id):
    if user_id not in bf:
        return None  # 一定不存在
    return mc.get(f'user:{user_id}') or db.query(...)

安全建议

  1. 不要在公网直接暴露 Memcached
  2. 使用防火墙限制访问
  3. 启用 SASL 认证
  4. 使用 TLS 加密传输
  5. 定期更新到最新版本
  6. 监控异常访问模式

容量规划

计算所需内存:

总内存 = 平均对象大小 × 对象数量 × 1.3(开销因子)

示例:

  • 100万个对象
  • 平均每个 1KB
  • 所需内存 ≈ 1GB × 1.3 = 1.3GB

附录

错误代码

  • ERROR: 一般错误
  • CLIENT_ERROR: 客户端错误
  • SERVER_ERROR: 服务器错误
  • NOT_FOUND: 键不存在
  • EXISTS: 键已存在(add 命令)
  • NOT_STORED: 未存储(如 replace 不存在的键)
