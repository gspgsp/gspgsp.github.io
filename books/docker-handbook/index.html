Docker 使用手册

Docker 简介

Docker 是一个开源的容器化平台,可以让开发者将应用及其依赖打包到轻量级、可移植的容器中,实现"一次构建,到处运行"。

主要优势

  • 环境一致性: 开发、测试、生产环境完全一致
  • 快速部署: 秒级启动容器
  • 资源隔离: 容器之间相互隔离,互不影响
  • 轻量高效: 相比虚拟机占用更少资源
  • 版本控制: 镜像支持版本管理和回滚

安装 Docker

Linux (Ubuntu/Debian)

# 更新软件包索引
sudo apt-get update

# 安装依赖
sudo apt-get install ca-certificates curl gnupg lsb-release

# 添加 Docker 官方 GPG 密钥
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 设置仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker Engine
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 验证安装
sudo docker run hello-world

macOS

# 使用 Homebrew 安装
brew install --cask docker

# 或下载 Docker Desktop
# 访问: https://www.docker.com/products/docker-desktop

Windows

下载并安装 Docker Desktop for Windows: https://www.docker.com/products/docker-desktop

配置用户权限 (Linux)

# 将当前用户添加到 docker 组
sudo usermod -aG docker $USER

# 重新登录或执行
newgrp docker

Docker 核心概念

镜像 (Image)

镜像是一个只读的模板,包含运行应用所需的代码、运行时、库、环境变量和配置文件。

容器 (Container)

容器是镜像的运行实例,是一个独立运行的应用环境。

仓库 (Registry)

用于存储和分发 Docker 镜像的服务,如 Docker Hub。

Dockerfile

用于构建镜像的文本文件,包含一系列构建指令。

镜像管理

搜索镜像

# 在 Docker Hub 搜索镜像
docker search nginx

# 搜索官方镜像
docker search --filter "is-official=true" nginx

拉取镜像

# 拉取最新版本
docker pull nginx

# 拉取指定版本
docker pull nginx:1.25.3

# 拉取指定平台镜像
docker pull --platform linux/amd64 nginx

查看本地镜像

# 列出所有镜像
docker images

# 查看镜像详细信息
docker image inspect nginx

# 查看镜像历史
docker history nginx

删除镜像

# 删除指定镜像
docker rmi nginx:latest

# 删除多个镜像
docker rmi nginx:1.25 nginx:1.24

# 删除所有未使用的镜像
docker image prune -a

# 强制删除镜像
docker rmi -f image_id

导入导出镜像

# 导出镜像为 tar 文件
docker save -o nginx.tar nginx:latest

# 从 tar 文件导入镜像
docker load -i nginx.tar

# 导出容器为镜像
docker export container_id > container.tar

# 从导出的容器创建镜像
docker import container.tar myimage:tag

标记镜像

# 为镜像添加新标签
docker tag nginx:latest myregistry.com/nginx:v1.0

# 推送到私有仓库
docker push myregistry.com/nginx:v1.0

容器管理

创建和运行容器

# 运行容器(前台)
docker run nginx

# 运行容器(后台)
docker run -d nginx

# 运行容器并命名
docker run -d --name my-nginx nginx

# 映射端口
docker run -d -p 8080:80 nginx

# 映射所有端口
docker run -d -P nginx

# 设置环境变量
docker run -d -e "ENV=production" nginx

# 挂载数据卷
docker run -d -v /host/path:/container/path nginx

# 限制资源
docker run -d --memory="512m" --cpus="1.5" nginx

# 重启策略
docker run -d --restart=always nginx

容器生命周期管理

# 启动容器
docker start container_id

# 停止容器
docker stop container_id

# 重启容器
docker restart container_id

# 暂停容器
docker pause container_id

# 恢复容器
docker unpause container_id

# 删除容器
docker rm container_id

# 强制删除运行中的容器
docker rm -f container_id

# 删除所有停止的容器
docker container prune

查看容器信息

# 列出运行中的容器
docker ps

# 列出所有容器(包括停止的)
docker ps -a

# 查看容器详细信息
docker inspect container_id

# 查看容器日志
docker logs container_id

# 实时查看日志
docker logs -f container_id

# 查看最近 100 行日志
docker logs --tail 100 container_id

# 查看容器资源使用情况
docker stats

# 查看容器进程
docker top container_id

进入容器

# 交互式进入容器
docker exec -it container_id /bin/bash

# 以 root 用户进入
docker exec -it -u root container_id /bin/bash

# 执行单个命令
docker exec container_id ls -la

# 使用 attach (不推荐,退出会停止容器)
docker attach container_id

容器与主机间的文件操作

# 从容器复制文件到主机
docker cp container_id:/path/to/file /host/path

# 从主机复制文件到容器
docker cp /host/path/file container_id:/path/to/

容器提交为镜像

# 将容器当前状态保存为新镜像
docker commit container_id myimage:v1.0

# 添加提交信息
docker commit -m "Added nginx config" -a "Author Name" container_id myimage:v1.0

网络配置

网络模式

  • bridge: 默认模式,容器连接到虚拟网桥
  • host: 容器使用主机网络
  • none: 容器没有网络
  • container: 容器共享另一个容器的网络

网络管理

# 列出所有网络
docker network ls

# 创建网络
docker network create my-network

# 创建指定子网的网络
docker network create --subnet=172.18.0.0/16 my-network

# 查看网络详情
docker network inspect my-network

# 连接容器到网络
docker network connect my-network container_id

# 断开容器网络连接
docker network disconnect my-network container_id

# 删除网络
docker network rm my-network

# 删除所有未使用的网络
docker network prune

使用自定义网络运行容器

# 运行容器并连接到指定网络
docker run -d --name web --network my-network nginx

# 同一网络内容器可以通过容器名互相访问
docker run -d --name app --network my-network myapp

数据卷管理

数据卷类型

  • 命名卷 (Named Volume): Docker 管理的持久化存储
  • 绑定挂载 (Bind Mount): 挂载主机目录到容器
  • tmpfs 挂载: 挂载到内存,容器停止后数据丢失

数据卷操作

# 创建数据卷
docker volume create my-volume

# 列出所有数据卷
docker volume ls

# 查看数据卷详情
docker volume inspect my-volume

# 删除数据卷
docker volume rm my-volume

# 删除所有未使用的数据卷
docker volume prune

使用数据卷

# 使用命名卷
docker run -d -v my-volume:/data nginx

# 使用绑定挂载(绝对路径)
docker run -d -v /host/path:/container/path nginx

# 只读挂载
docker run -d -v my-volume:/data:ro nginx

# 使用 tmpfs
docker run -d --tmpfs /app/cache nginx

# 使用 --mount(更明确的语法)
docker run -d \
  --mount type=volume,source=my-volume,target=/data \
  nginx

数据卷备份和恢复

# 备份数据卷
docker run --rm -v my-volume:/data -v $(pwd):/backup ubuntu tar czf /backup/backup.tar.gz /data

# 恢复数据卷
docker run --rm -v my-volume:/data -v $(pwd):/backup ubuntu tar xzf /backup/backup.tar.gz -C /

Docker Compose

Docker Compose 是用于定义和运行多容器 Docker 应用的工具。

安装 Docker Compose

# Docker Desktop 自带 Compose V2
# Linux 上安装
sudo apt-get install docker-compose-plugin

# 验证安装
docker compose version

核心用途

  • 启动几个服务（php、nginx、mysql、redis）
  • 端口如何映射
  • 环境变量
  • 容器之间如何通信
  • 挂载 volume
  • 依赖关系（depends_on）

docker-compose.yaml 示例

version: "3.9" # 3.9表示这个 docker-compose 文件，遵循 3.9 版本的语法规范, 对于Compose v2 可以不用写这个version信息, 之前很多是3.8(Compose v1)

services:
   php:
      build: . # 先用 从php的Dockerfile 构建镜像, 并且这里使用默认的dockerfile: Dockerfile(所以不用显示写)， 如果需要使用其它名称的可以显示定义，如 dockerfile: Dockerfile.dev
      volumes:
         - .:/app
      networks:
         - app

   nginx:
      image: nginx:alpine # 直接用官方镜像，不自己构建
      ports:
         - "8080:80"
      volumes:
         - ./docker/nginx.conf:/etc/nginx/nginx.conf
      depends_on:
         - php
      networks:
         - app

   mysql:
      image: mysql:8.0 # 直接用官方镜像，不自己构建
      environment:
         MYSQL_ROOT_PASSWORD: root
         MYSQL_DATABASE: app
      volumes:
         - mysql_data:/var/lib/mysql
      networks:
         - app

networks:
   app:

volumes:
   mysql_data:

Compose 常用命令(v2版本-go版-docker CLI 插件(plugin)，v1版-python版-独立工具使用的是: docker-compose up 等)

# 启动所有服务(后台)
docker compose up -d

# 启动并重新构建镜像
docker compose up -d --build

# 查看服务状态
docker compose ps

# 查看服务日志
docker compose logs

# 实时查看日志
docker compose logs -f

# 查看特定服务日志
docker compose logs web

# 停止服务
docker compose stop

# 停止并删除容器
docker compose down

# 停止并删除容器、网络、数据卷
docker compose down -v

# 重启服务
docker compose restart

# 执行命令
docker compose exec web bash

# 扩展服务
docker compose up -d --scale web=3

# 验证配置文件
docker compose config

# 拉取所有镜像
docker compose pull

Dockerfile 编写

Dockerfile 基础结构

# 指定基础镜像
FROM node:18-alpine

# 设置工作目录
WORKDIR /app

# 设置环境变量
ENV NODE_ENV=production

# 复制依赖文件
COPY package*.json ./

# 安装依赖
RUN npm ci --only=production

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 3000

# 设置用户
USER node

# 启动命令
CMD ["node", "server.js"]

核心用途

  • 基于哪个基础镜像（php、node、nginx…）
  • 安装什么系统依赖 / 扩展
  • 拷贝代码
  • 如何编译 / build
  • 容器启动时执行什么命令

常用指令详解

FROM

# 使用官方基础镜像
FROM ubuntu:22.04

# 使用多阶段构建
FROM golang:1.21 AS builder

RUN

# 执行命令并创建新层
RUN apt-get update && apt-get install -y \
    package1 \
    package2 \
    && rm -rf /var/lib/apt/lists/*

# 使用 && 连接命令减少层数
RUN command1 && command2 && command3

COPY vs ADD

# COPY: 简单复制文件(推荐)
COPY src/ /app/

# ADD: 支持 URL 和自动解压 tar
ADD https://example.com/file.tar.gz /app/
ADD archive.tar.gz /app/

ENV

# 设置环境变量
ENV APP_HOME=/app \
    PORT=8080

ARG

# 构建时参数
ARG VERSION=1.0
RUN echo "Building version ${VERSION}"

# 使用: docker build --build-arg VERSION=2.0 .

WORKDIR

# 设置工作目录(自动创建)
WORKDIR /app

EXPOSE

# 声明容器监听端口(仅文档作用)
EXPOSE 80 443

VOLUME

# 声明挂载点
VOLUME /data

USER

# 切换用户
USER appuser

CMD vs ENTRYPOINT

# CMD: 默认命令(可被覆盖)
CMD ["nginx", "-g", "daemon off;"]

# ENTRYPOINT: 容器入口点(不易覆盖)
ENTRYPOINT ["docker-entrypoint.sh"]

# 组合使用
ENTRYPOINT ["python"]
CMD ["app.py"]

多阶段构建示例

# 构建阶段
FROM golang:1.21 AS builder
WORKDIR /app
COPY go.* ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -o myapp

# 运行阶段
FROM alpine:latest
RUN apk --no-cache add ca-certificates
WORKDIR /root/
COPY --from=builder /app/myapp .
CMD ["./myapp"]

构建镜像

# 基本构建
docker build -t myapp:v1.0 .

# 指定 Dockerfile
docker build -f Dockerfile.prod -t myapp:prod .

# 使用构建参数
docker build --build-arg VERSION=2.0 -t myapp:v2.0 .

# 不使用缓存
docker build --no-cache -t myapp:v1.0 .

# 指定平台
docker build --platform linux/amd64 -t myapp:v1.0 .

# 停止所有运行中的容器

最佳实践

镜像构建最佳实践

  1. 使用合适的基础镜像

    • 优先选择官方镜像
    • 使用 alpine 版本减小镜像体积
    • 使用特定版本标签而非 latest

  2. 优化层缓存

    • 将变化频率低的指令放在前面
    • 先复制依赖文件,再复制代码
    • 合并 RUN 指令减少层数

  3. 减小镜像体积

    • 使用多阶段构建
    • 清理临时文件和缓存
    • 使用 .dockerignore 排除无用文件

  4. 安全性

    • 不要在镜像中存储敏感信息
    • 使用非 root 用户运行应用
    • 定期更新基础镜像
    • 扫描镜像漏洞

容器运行最佳实践

  1. 资源限制
docker run -d \
  --memory="512m" \
  --memory-swap="1g" \
  --cpus="1.5" \
  --pids-limit=100 \
  myapp
  1. 健康检查
HEALTHCHECK --interval=30s --timeout=3s \
  CMD curl -f http://localhost/ || exit 1
  1. 日志管理
# 限制日志大小
docker run -d \
  --log-opt max-size=10m \
  --log-opt max-file=3 \
  myapp
  1. 重启策略
# 总是重启
docker run -d --restart=always myapp

# 失败时重启
docker run -d --restart=on-failure:5 myapp

Docker Compose 最佳实践

  1. 环境变量管理
# 使用 .env 文件
services:
  web:
    environment:
      - DB_HOST=${DB_HOST}
      - DB_PORT=${DB_PORT}
  1. 服务依赖
services:
  web:
    depends_on:
      db:
        condition: service_healthy
  db:
    healthcheck:
      test: ["CMD", "pg_isready"]
  1. 开发与生产配置分离
# 开发环境
docker compose -f compose.yaml -f compose.dev.yaml up

# 生产环境
docker compose -f compose.yaml -f compose.prod.yaml up

性能优化

  1. 使用数据卷而非绑定挂载
  2. 合理配置网络模式
  3. 使用 BuildKit 加速构建
DOCKER_BUILDKIT=1 docker build -t myapp .
  1. 配置镜像加速器
{
  "registry-mirrors": [
    "https://mirror.gcr.io"
  ]
}

监控和日志

  1. 使用 Prometheus 监控容器
  2. 集中化日志收集(ELK/EFK)
  3. 使用 docker stats 实时监控资源
  4. 配置日志驱动(json-file, syslog, journald)

安全建议

  1. 定期更新 Docker 和镜像
  2. 使用内容信任验证镜像
  3. 限制容器权限
docker run --cap-drop=ALL --cap-add=NET_BIND_SERVICE myapp
  1. 使用只读文件系统
docker run --read-only --tmpfs /tmp myapp
  1. 启用 Docker 安全扫描
docker scan myimage:tag

# 查看端口占用
lsof -i :8080

# 清理未使用资源

网络连接问题

# 检查容器网络配置

docker exec container_id ping google.com
