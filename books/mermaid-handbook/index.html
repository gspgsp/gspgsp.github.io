一、Mermaid 简介

Mermaid 是一款基于 JavaScript 的图表绘制工具，使用类似 Markdown 的文本语法来创建和修改图表。它的核心优势：

  • 纯文本描述图表：图表内容可以像代码一样进行版本管理（Git diff 友好）。
  • 无需设计工具：不用 Visio、draw.io，写几行文本即可生成流程图、时序图等。
  • 生态集成广泛：GitHub、GitLab、Notion、VuePress、VitePress、Typora 等均已原生或通过插件支持。

官方文档：https://mermaid.js.org/

二、在 VuePress2 中集成 Mermaid

VuePress2 本身不内置 Mermaid 渲染，需要借助插件。推荐两种主流方案：

方案 A：vuepress-plugin-md-enhance（推荐，官方生态维护）

该插件是 vuepress-theme-hope 生态下的 Markdown 增强插件，内置 Mermaid 支持。

安装：

npm install -D vuepress-plugin-md-enhance@next

配置（.vuepress/config.ts）：

import { defineUserConfig } from "vuepress";
import { mdEnhancePlugin } from "vuepress-plugin-md-enhance";

export default defineUserConfig({
  bundler: "@vuepress/bundler-vite", // 或 webpack
  plugins: [
    mdEnhancePlugin({
      mermaid: true, // 开启 mermaid 支持
    }),
  ],
});

启用后，直接在 Markdown 中写 ```mermaid 代码块即可自动渲染。

方案 B：手动集成 mermaid 包 + 自定义组件

如果不想引入整套增强插件，可以自己封装一个 Vue 组件：

npm install mermaid

.vuepress/components/Mermaid.vue：

<template>
  <div class="mermaid" ref="el"></div>
</template>

<script setup>
import { onMounted, ref } from "vue";
import mermaid from "mermaid";

const props = defineProps({ code: String });
const el = ref(null);

onMounted(async () => {
  mermaid.initialize({ startOnLoad: false, theme: "default" });
  const { svg } = await mermaid.render("graph-" + Date.now(), props.code);
  el.value.innerHTML = svg;
});
</script>

然后在 Markdown 中通过 <Mermaid code="..." /> 调用。方案 A 更省心，本手册后续 Demo 均按方案 A（```mermaid 代码块）书写，直接复制到你的 .md 文件即可。

注意事项

  • VuePress2 默认使用 Vite 打包时兼容性较好；若使用 Webpack，请确认插件版本支持。
  • Mermaid 图表在 深色模式 下可能需要额外配置主题切换，见第十六节。
  • 大型图表建议开启 securityLevel: "loose" 以支持点击跳转等交互（谨慎使用，避免 XSS 风险）。

四、流程图 Flowchart

4.1 方向控制

flowchart 后可跟方向：TB（上到下，默认）、BT（下到上）、LR（左到右）、RL（右到左）。

Demo：审批流程

渲染效果：

源码：

```mermaid

    Start([提交申请]) --> Check{金额是否 > 10000?}
    Check -- 是 --> Manager[经理审批]
    Check -- 否 --> Auto[系统自动通过]
    Manager --> Decision{经理是否同意?}
    Decision -- 同意 --> Finance[财务复核]
    Decision -- 拒绝 --> Reject([驳回申请])
    Finance --> End([流程结束])
    Auto --> End
```

五、时序图 Sequence Diagram

Demo：用户登录时序

渲染效果：

源码：

```mermaid
sequenceDiagram
    autonumber
    participant U as 用户
    participant F as 前端
    participant S as 后端服务
    participant D as 数据库

    U->>F: 输入账号密码
    F->>S: POST /login
    activate S
    S->>D: 查询用户信息
    D-->>S: 返回用户记录
    alt 密码正确
        S-->>F: 200 返回 Token
        F-->>U: 跳转首页
    else 密码错误
        S-->>F: 401 认证失败
        F-->>U: 提示错误
    end
    deactivate S
```

常用元素说明：

  • participant 声明参与者，as 可设置别名。
  • ->> 实线箭头（同步请求），-->> 虚线箭头（返回/异步）。
  • activate / deactivate 标注生命线激活区间。
  • alt / else / end 条件分支；loop / end 循环；par / and / end 并行。
  • Note over A,B: 说明文字 添加备注。

渲染效果：

源码：

```mermaid
sequenceDiagram
    participant A as 客户端
    participant B as 服务端
    loop 每 5 秒轮询
        A->>B: 心跳请求
        B-->>A: 心跳响应
    end
    Note over A,B: 保持长连接
```

八、实体关系图 ER Diagram

Demo：博客系统数据模型

```mermaid

    USER ||--o{ COMMENT : 撰写

    USER {
```

关系基数符号：|| 恰好一个，o| 零或一个，}o 零或多个，}| 一个或多个。

十六、样式与主题定制

16.1 全局主题（在配置中设置）

mdEnhancePlugin({
  mermaid: {
    theme: "dark", // default | forest | dark | neutral
  },
});

16.2 单个图表内联样式（%%{init}%% 指令）

渲染效果：

源码：

```mermaid
%%{init: {'theme':'base', 'themeVariables': {'primaryColor':'#ff6b6b','primaryTextColor':'#fff','lineColor':'#333'}}}%%
flowchart LR
    A[自定义颜色节点] --> B[另一节点]
```

16.3 使用 classDef 自定义节点样式

渲染效果：

源码：

```mermaid
flowchart LR
    A[成功] --> B[失败]
    classDef ok fill:#d4edda,stroke:#28a745,color:#155724;
    classDef fail fill:#f8d7da,stroke:#dc3545,color:#721c24;
    class A ok
    class B fail
```

16.4 深色模式适配（VuePress2）

若主题支持暗色模式切换，建议监听 html 的 class 变化，动态调用 mermaid.initialize 并重新渲染，或直接使用 theme: 'neutral' 保证明暗两种模式下都可读。

十七、常见问题排查

问题可能原因解决方法
图表不渲染，显示原始文本插件未启用 mermaid: true检查 .vuepress/config.ts 配置
中文节点文字乱码/换行异常节点文本包含特殊符号未加引号用 A["带有(特殊)符号的文字"] 包裹
图表在移动端被截断图表过宽使用 flowchart TD（纵向）或拆分子图
构建时报错 Maximum call stack版本冲突（mermaid 与插件不匹配）锁定 mermaid 与插件的兼容版本
点击跳转链接无效securityLevel 未设置为 loose在 init 配置中设置 securityLevel: 'loose'
SSR 构建报错 document is not definedMermaid 依赖浏览器环境确认插件已做 ClientOnly 包裹（vuepress-plugin-md-enhance 已内置处理）

十八、最佳实践建议

  1. 图表拆分：单张图表节点数建议不超过 20 个，过复杂的流程拆成多张子图，便于维护和阅读。
  2. 命名规范：节点 ID 使用英文简写（如 A, Start, CheckAuth），显示文本用中文，避免中文 ID 导致的解析问题。
  3. 版本管理：Mermaid 源码是纯文本，天然适合 Git diff review，建议将复杂图表的变更放在独立的 commit 中。
  4. 配合注释：在图表上方用 Markdown 文字简要说明图表要表达的核心逻辑，图表本身仅做可视化补充。
  5. 锁定版本：mermaid 更新较快，语法偶有 breaking change，生产文档站点建议锁定版本号，升级前先在本地预览验证。
  6. 导出静态图（可选）：如需在图表无法渲染的环境（如打印 PDF）展示，可用 mermaid-cli（mmdc）预先导出为 SVG/PNG 作为兜底。
npm install -g @mermaid-js/mermaid-cli
mmdc -i input.mmd -o output.svg
