MongoDB 使用手册

简介

MongoDB 是一个基于分布式文件存储的 NoSQL 数据库系统,由 C++ 编写。它支持丰富的查询语言,提供高性能、高可用性和易扩展性。

主要特点:

  • 面向文档存储,使用 BSON(Binary JSON)格式
  • 支持动态模式,无需预定义表结构
  • 强大的查询和聚合功能
  • 自动分片和副本集支持
  • 支持多文档 ACID 事务

适用场景:

  • 内容管理系统
  • 实时分析
  • 物联网数据存储
  • 移动应用后端
  • 游戏数据存储

安装与配置

安装 MongoDB

Linux (Ubuntu/Debian):

# 导入公钥
wget -qO - https://www.mongodb.org/static/pgp/server-7.0.asc | sudo apt-key add -

# 添加源
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list

# 安装
sudo apt-get update
sudo apt-get install -y mongodb-org

# 启动服务
sudo systemctl start mongod
sudo systemctl enable mongod

macOS:

# 使用 Homebrew
brew tap mongodb/brew
brew install mongodb-community@7.0

# 启动服务
brew services start mongodb-community@7.0

Windows: 下载 MSI 安装包从 MongoDB 官网,按照向导安装。

配置文件

MongoDB 配置文件通常位于 /etc/mongod.conf (Linux) 或 C:\Program Files\MongoDB\Server\7.0\bin\mongod.cfg (Windows)。

基本配置示例:

storage:
  dbPath: /var/lib/mongodb
  journal:
    enabled: true

systemLog:
  destination: file
  path: /var/log/mongodb/mongod.log
  logAppend: true

net:
  port: 27017
  bindIp: 127.0.0.1

security:
  authorization: enabled

连接 MongoDB

使用 MongoDB Shell (mongosh):

mongosh
# 或连接到远程服务器
mongosh "mongodb://username:password@hostname:27017"

基础概念

数据模型

  • 数据库 (Database): 数据库的容器,每个数据库都有独立的文件系统
  • 集合 (Collection): 类似关系数据库中的表,存储文档的集合
  • 文档 (Document): 基本数据单元,类似 JSON 对象,使用 BSON 格式存储
  • 字段 (Field): 文档中的键值对

文档结构示例

{
  "_id": ObjectId("507f1f77bcf86cd799439011"),
  "name": "张三",
  "age": 28,
  "email": "zhangsan@example.com",
  "address": {
    "city": "北京",
    "district": "朝阳区"
  },
  "hobbies": ["阅读", "旅游", "摄影"],
  "createdAt": ISODate("2024-01-15T08:30:00Z")
}

查看数据库

// 显示所有数据库
show dbs

// 显示当前使用的数据库
db

// 切换到指定数据库(如果不存在则创建)
use myDatabase

删除数据库

// 删除当前数据库
db.dropDatabase()

集合操作

创建集合

// 显式创建集合
db.createCollection("users")

// 创建带选项的集合
db.createCollection("orders", {
  capped: true,        // 固定大小集合
  size: 5242880,       // 最大字节数
  max: 5000            // 最大文档数
})

// 隐式创建(插入文档时自动创建)
db.products.insertOne({name: "商品1"})

查看集合

// 显示所有集合
show collections

// 获取集合统计信息
db.users.stats()

重命名集合

db.users.renameCollection("customers")

删除集合

db.users.drop()

文档CRUD操作

插入文档

插入单个文档:

db.users.insertOne({
  name: "李四",
  age: 25,
  email: "lisi@example.com",
  createdAt: new Date()
})

插入多个文档:

db.users.insertMany([
  {name: "王五", age: 30, email: "wangwu@example.com"},
  {name: "赵六", age: 22, email: "zhaoliu@example.com"},
  {name: "孙七", age: 35, email: "sunqi@example.com"}
])

查询文档

查询所有文档:

db.users.find()

条件查询:

// 查询单个文档
db.users.findOne({name: "李四"})

// 按条件查询
db.users.find({age: {$gte: 25}})

// 返回指定字段(投影)
db.users.find({}, {name: 1, email: 1, _id: 0})

// 限制返回数量
db.users.find().limit(10)

// 跳过指定数量
db.users.find().skip(5).limit(10)

// 排序
db.users.find().sort({age: -1})  // -1降序, 1升序

更新文档

更新单个文档:

db.users.updateOne(
  {name: "李四"},
  {$set: {age: 26, email: "lisi_new@example.com"}}
)

更新多个文档:

db.users.updateMany(
  {age: {$lt: 30}},
  {$inc: {age: 1}}  // 年龄加1
)

替换文档:

db.users.replaceOne(
  {name: "李四"},
  {name: "李四", age: 27, email: "lisi@example.com", updatedAt: new Date()}
)

常用更新操作符:

  • $set: 设置字段值
  • $unset: 删除字段
  • $inc: 增加数值
  • $push: 向数组添加元素
  • $pull: 从数组删除元素
  • $addToSet: 向数组添加唯一元素

删除文档

删除单个文档:

db.users.deleteOne({name: "李四"})

删除多个文档:

db.users.deleteMany({age: {$lt: 25}})

删除所有文档:

db.users.deleteMany({})

查询操作

比较操作符

// 等于
db.users.find({age: 25})

// 大于 ($gt), 大于等于 ($gte)
db.users.find({age: {$gt: 25}})
db.users.find({age: {$gte: 25}})

// 小于 ($lt), 小于等于 ($lte)
db.users.find({age: {$lt: 30}})
db.users.find({age: {$lte: 30}})

// 不等于 ($ne)
db.users.find({status: {$ne: "inactive"}})

// 在范围内 ($in)
db.users.find({age: {$in: [25, 30, 35]}})

// 不在范围内 ($nin)
db.users.find({age: {$nin: [25, 30, 35]}})

逻辑操作符

// AND (隐式)
db.users.find({age: {$gte: 25}, status: "active"})

// OR
db.users.find({
  $or: [
    {age: {$lt: 25}},
    {age: {$gt: 35}}
  ]
})

// NOT
db.users.find({age: {$not: {$gte: 30}}})

// NOR
db.users.find({
  $nor: [
    {age: {$lt: 25}},
    {status: "inactive"}
  ]
})

元素操作符

// 字段存在
db.users.find({email: {$exists: true}})

// 类型检查
db.users.find({age: {$type: "number"}})

数组操作符

// 包含指定元素
db.users.find({hobbies: "阅读"})

// 包含所有指定元素
db.users.find({hobbies: {$all: ["阅读", "旅游"]}})

// 数组大小
db.users.find({hobbies: {$size: 3}})

// 数组元素匹配
db.users.find({
  "scores": {
    $elemMatch: {$gte: 80, $lt: 90}
  }
})

正则表达式查询

// 包含特定字符串
db.users.find({name: {$regex: /张/}})

// 以特定字符串开头
db.users.find({email: {$regex: /^zhangsan/}})

// 不区分大小写
db.users.find({name: {$regex: /zhang/i}})

嵌套文档查询

// 精确匹配嵌套文档
db.users.find({
  address: {city: "北京", district: "朝阳区"}
})

// 查询嵌套字段
db.users.find({"address.city": "北京"})

索引

创建索引

单字段索引:

// 升序索引
db.users.createIndex({name: 1})

// 降序索引
db.users.createIndex({age: -1})

复合索引:

db.users.createIndex({name: 1, age: -1})

唯一索引:

db.users.createIndex({email: 1}, {unique: true})

稀疏索引:

// 只索引存在该字段的文档
db.users.createIndex({phone: 1}, {sparse: true})

文本索引:

db.articles.createIndex({content: "text"})

// 查询
db.articles.find({$text: {$search: "MongoDB 教程"}})

地理空间索引:

db.places.createIndex({location: "2dsphere"})

TTL索引(自动过期):

// 文档创建后3600秒自动删除
db.sessions.createIndex({createdAt: 1}, {expireAfterSeconds: 3600})

查看索引

// 查看集合的所有索引
db.users.getIndexes()

// 查看索引大小
db.users.totalIndexSize()

删除索引

// 删除指定索引
db.users.dropIndex("name_1")

// 删除所有索引(除了_id索引)
db.users.dropIndexes()

分析查询性能

// 使用 explain() 分析查询计划
db.users.find({age: {$gte: 25}}).explain("executionStats")

聚合操作

聚合管道提供强大的数据处理和分析功能。

基本聚合阶段

$match - 过滤文档:

db.orders.aggregate([
  {$match: {status: "completed"}}
])

$group - 分组统计:

db.orders.aggregate([
  {
    $group: {
      _id: "$customerId",
      totalAmount: {$sum: "$amount"},
      orderCount: {$sum: 1},
      avgAmount: {$avg: "$amount"}
    }
  }
])

$project - 字段投影:

db.users.aggregate([
  {
    $project: {
      name: 1,
      email: 1,
      ageGroup: {
        $cond: {
          if: {$gte: ["$age", 30]},
          then: "成年",
          else: "青年"
        }
      }
    }
  }
])

$sort - 排序:

db.orders.aggregate([
  {$sort: {amount: -1}}
])

$limit 和 $skip:

db.users.aggregate([
  {$sort: {age: -1}},
  {$skip: 10},
  {$limit: 5}
])

$lookup - 关联查询:

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "customerId",
      foreignField: "_id",
      as: "customerInfo"
    }
  }
])

$unwind - 展开数组:

db.users.aggregate([
  {$unwind: "$hobbies"}
])

聚合操作符

算术操作符:

  • $add, $subtract, $multiply, $divide
  • $mod, $abs, $ceil, $floor

字符串操作符:

  • $concat, $substr, $toLower, $toUpper
  • $split, $trim

日期操作符:

  • $year, $month, $dayOfMonth
  • $hour, $minute, $second

条件操作符:

  • $cond, $ifNull, $switch

聚合示例

计算每月销售额:

db.orders.aggregate([
  {
    $group: {
      _id: {
        year: {$year: "$orderDate"},
        month: {$month: "$orderDate"}
      },
      totalSales: {$sum: "$amount"},
      orderCount: {$sum: 1}
    }
  },
  {$sort: {"_id.year": -1, "_id.month": -1}}
])

用户消费排行榜:

db.orders.aggregate([
  {$match: {status: "completed"}},
  {
    $group: {
      _id: "$customerId",
      totalSpent: {$sum: "$amount"}
    }
  },
  {$sort: {totalSpent: -1}},
  {$limit: 10},
  {
    $lookup: {
      from: "customers",
      localField: "_id",
      foreignField: "_id",
      as: "customer"
    }
  },
  {$unwind: "$customer"},
  {
    $project: {
      customerName: "$customer.name",
      totalSpent: 1
    }
  }
])

事务

MongoDB 支持多文档 ACID 事务,确保数据一致性。

使用事务

const session = db.getMongo().startSession()

session.startTransaction()

try {
  const ordersCollection = session.getDatabase("mydb").orders
  const inventoryCollection = session.getDatabase("mydb").inventory
  
  // 创建订单
  ordersCollection.insertOne(
    {
      orderId: "ORD-001",
      productId: "PROD-001",
      quantity: 5,
      status: "pending"
    },
    {session}
  )
  
  // 减少库存
  inventoryCollection.updateOne(
    {productId: "PROD-001"},
    {$inc: {stock: -5}},
    {session}
  )
  
  // 提交事务
  session.commitTransaction()
  print("事务提交成功")
  
} catch (error) {
  // 回滚事务
  session.abortTransaction()
  print("事务回滚: " + error)
  
} finally {
  session.endSession()
}

事务注意事项

  • 事务只能在副本集或分片集群上使用
  • 事务有60秒的默认超时时间
  • 避免在事务中执行长时间运行的操作
  • 事务中的所有操作必须在同一个会话中

用户管理与安全

创建管理员用户

use admin

db.createUser({
  user: "admin",
  pwd: "securePassword123",
  roles: [
    {role: "userAdminAnyDatabase", db: "admin"},
    {role: "readWriteAnyDatabase", db: "admin"}
  ]
})

创建普通用户

use myDatabase

db.createUser({
  user: "appUser",
  pwd: "appPassword123",
  roles: [
    {role: "readWrite", db: "myDatabase"}
  ]
})

常用角色

数据库角色:

  • read: 读取权限
  • readWrite: 读写权限
  • dbAdmin: 数据库管理权限
  • dbOwner: 数据库所有者权限

集群角色:

  • clusterAdmin: 集群管理权限
  • clusterMonitor: 集群监控权限

全局角色:

  • readAnyDatabase: 读取所有数据库
  • readWriteAnyDatabase: 读写所有数据库
  • userAdminAnyDatabase: 用户管理权限
  • dbAdminAnyDatabase: 数据库管理权限

用户管理操作

// 查看当前数据库的用户
db.getUsers()

// 修改用户密码
db.changeUserPassword("appUser", "newPassword456")

// 授予角色
db.grantRolesToUser("appUser", [
  {role: "read", db: "anotherDB"}
])

// 撤销角色
db.revokeRolesFromUser("appUser", [
  {role: "read", db: "anotherDB"}
])

// 删除用户
db.dropUser("appUser")

启用访问控制

在配置文件中启用认证:

security:
  authorization: enabled

连接时使用认证:

mongosh "mongodb://username:password@localhost:27017/database?authSource=admin"

备份与恢复

使用 mongodump 备份

备份整个数据库:

mongodump --db=myDatabase --out=/backup/directory

备份特定集合:

mongodump --db=myDatabase --collection=users --out=/backup/directory

备份所有数据库:

mongodump --out=/backup/directory

带认证的备份:

mongodump --host=localhost --port=27017 \
  --username=admin --password=password \
  --authenticationDatabase=admin \
  --db=myDatabase --out=/backup/directory

使用 mongorestore 恢复

恢复数据库:

mongorestore --db=myDatabase /backup/directory/myDatabase

恢复特定集合:

mongorestore --db=myDatabase --collection=users \
  /backup/directory/myDatabase/users.bson

恢复所有数据库:

mongorestore /backup/directory

删除现有数据后恢复:

mongorestore --drop --db=myDatabase /backup/directory/myDatabase

导出和导入 JSON/CSV

导出为JSON:

mongoexport --db=myDatabase --collection=users --out=users.json

导出为CSV:

mongoexport --db=myDatabase --collection=users \
  --type=csv --fields=name,email,age --out=users.csv

导入JSON:

mongoimport --db=myDatabase --collection=users --file=users.json

导入CSV:

mongoimport --db=myDatabase --collection=users \
  --type=csv --headerline --file=users.csv

性能优化

索引优化

  • 为频繁查询的字段创建索引
  • 使用复合索引优化多字段查询
  • 定期检查和删除未使用的索引
  • 使用覆盖查询(查询字段全部在索引中)

检查慢查询:

// 启用慢查询日志
db.setProfilingLevel(1, {slowms: 100})

// 查看慢查询
db.system.profile.find().sort({ts: -1}).limit(10)

查询优化

  • 使用投影只返回需要的字段
  • 合理使用 limit() 限制返回数量
  • 避免使用 $where 和 JavaScript 表达式
  • 使用 explain() 分析查询计划

文档设计优化

  • 考虑嵌入式文档 vs 引用
  • 避免文档过大(BSON限制16MB)
  • 合理使用数组,避免无限增长
  • 考虑数据访问模式设计schema

连接池配置

// 设置连接池大小
const client = new MongoClient(uri, {
  maxPoolSize: 50,
  minPoolSize: 10,
  maxIdleTimeMS: 30000
})

硬件和配置优化

  • 使用SSD存储提升IO性能
  • 增加RAM以支持更多数据在内存中
  • 配置适当的WiredTiger缓存大小
  • 启用压缩节省存储空间

配置WiredTiger引擎:

storage:
  engine: wiredTiger
  wiredTiger:
    engineConfig:
      cacheSizeGB: 2
    collectionConfig:
      blockCompressor: snappy
